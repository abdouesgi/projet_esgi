# Guide de contribution au Gestionnaire de tâches

Nous sommes ravis que vous souhaitiez contribuer au Gestionnaire de tâches ! Veuillez suivre ces étapes pour nous aider à améliorer notre projet.

## Signaler des problèmes

Si vous trouvez un bogue ou souhaitez suggérer une nouvelle fonctionnalité, veuillez ouvrir une issue sur notre [page des problèmes](https://github.com/votre-utilisateur/gestionnaire-de-taches/issues).

## Contribuer au code

Si vous souhaitez contribuer directement au code, suivez ces étapes :

1. **Fork** du projet vers votre propre compte GitHub.
2. **Clonez** votre fork localement.
3. Créez une **branche** pour travailler sur votre modification (par exemple, `git checkout -b fonctionnalite-ma-tache`).
4. Faites vos modifications et **commitez**-les (par exemple, `git commit -m "Ajout de la fonctionnalité Ma Tâche"`).
5. **Poussez** vos modifications vers votre fork sur GitHub (par exemple, `git push origin fonctionnalite-ma-tache`).
6. Ouvrez une **demande de fusion (Pull Request)** sur notre page principale du projet.

## Style de code

Nous suivons des normes de style de code strictes. Assurez-vous de respecter ces normes lors de la rédaction de votre code. Vous pouvez trouver des informations détaillées dans notre [guide de style de code](style-guide.md).

## Tests

Veuillez ajouter des tests appropriés pour toutes les modifications de code que vous apportez. Les tests aident à maintenir la qualité du code et à éviter les régressions.

## Remerciements

Merci de contribuer au Gestionnaire de tâches ! Votre aide est grandement appréciée.

## Licence

En contribuant à ce projet, vous acceptez que votre travail soit distribué sous la [licence MIT](LICENSE) du projet.
